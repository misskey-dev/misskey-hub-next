# 通过创建应用来获取访问令牌的方法（旧方法）

这里将说明在引入 [MiAuth](./miauth.md)（v12.27.0）和 [OAuth](./oauth.md)（v2023.9.0）之前的传统访问令牌获取方法。  
低于 12.27.0 版本的服务器必须使用此传统方式。

## 1. 创建应用程序

向 [`app/create` 端点](endpoints/app/create.html)发送信息，以获取 `appSecret`。

```json
{
    // 应用程序名称
    "name": "test",
    // 应用程序说明
    "description": "my test application",
    // 应用程序权限
    "permission": ["write:notes"]
}
```

此时，如果在请求中包含您自定义的 `callbackUrl`，当接下来的访问授权操作完成后，系统会将 `token` 包含在 URL 参数中并回调至该 URL。

## 2. 让用户进行认证

将 `appSecret` 通过 POST 发送至 [`auth/session/generate` 端点](endpoints/auth/session/generate)。

```json
{
    "appSecret": "fAb12cD34Ef56gH78Ij16kL32Mn64oPf"
}
```

系统会返回 `token`（这里假设为 `798b9f6e-248d-43a7-a919-fabc664027f1`）和 `url`。请先使用 Web 浏览器访问此 url，并选择“允许访问”。

## 3. 查询 accessToken

确认步骤 2 已完成后，将 `appSecret` 和刚才获取的 `token` 通过 POST 发送至 [`auth/session/userkey` 端点](endpoints/auth/session/userkey)。

```json
{
  "appSecret": "fAb12cD34Ef56gH78Ij16kL32Mn64oPf",
  "token": "798b9f6e-248d-43a7-a919-fabc664027f1"
}
```

在这里获取的字符串称为 `accessToken`。请注意，`accessToken` 只能获取一次。

## 4. 生成 i

如果是使用 Node.js，可以通过以下代码生成 `i`。这与在设置页面中直接获取的格式不同，这里生成的 `i` 会是 64 位的十六进制字符串。

```javascript
const crypto = require("crypto")
const i = crypto.createHash("sha256")
    .update(accessToken + appSecret, "utf8")
    .digest("hex")
console.log(i)
```

## 5. 实际进行测试

```javascript
fetch("https://misskey.example/api/notes/create", {
    method: 'POST',
    body: JSON.stringify({
        i: "/* 在此输入生成的 i */",
        text: "Hello Misskey API World with My Application!"
    }),
    headers: {
        'Content-Type': 'application/json',
    },
	credentials: 'omit',
});
```

# 通过 MiAuth 方式获取访问令牌 (Access Token)

要获取使用应用程序的用户（以下简称“用户”）的访问令牌，请按照以下步骤请求发行。

:::tip

以下说明的方法称为 MiAuth，这是一种无需创建应用即可即时发行访问令牌的方法。

- [此外，还有一种目前使用更为广泛的通过 OAuth 方式获取访问令牌的方法。](./oauth.md)
- [也有通过创建应用来获取访问令牌的方法（旧方法）。](./app.md)

:::

## 步骤 1

生成UUID。以下将其称为**会话 ID** (Session ID)。

:::danger

请务必每次都生成新的会话 ID，请勿重复使用。

:::

## 步骤 2

在用户的浏览器中显示应用程序认证表单。可以通过以下格式的 URL 打开认证表单：

```
https://{host}/miauth/{session}
```

在此：

- `{host}` 部分替换为用户服务器的主机名。通常主机名由用户输入。
- `{session}` 部分替换为会话 ID。

此外，您可以在 URL 中设置几个查询参数作为选项：

| 名称           | 说明                                                                   |
| ------------ | -------------------------------------------------------------------- |
| `name`       | 应用程序名称。                                                              |
| `icon`       | 应用程序的图标图片 URL。                                                       |
| `callback`   | 认证完成后重定向的 URL。<br>重定向时，会附带名为 session 的查询参数，其值为会话 ID。                 |
| `permission` | 应用程序请求的权限。<br>请将请求的权限以半角逗号 `,` 分隔列出。权限列表可以在[此处](../permission.md)确认。 |

:::tip{label='例'}

```
https://misskey.example/miauth/c1f6d42b-468b-4fd2-8274-e58abdedef6f?name=MyApp&callback=https%3A%2F%2Fmyapp.example.com%2Fcallback&permission=write:notes,write:following,read:drive
```

:::

## 步骤 3

用户允许应用程序访问后，向以下格式的 URL 发送 POST 请求，即可在响应中获得包含访问令牌的 JSON 数据。

```
https://{host}/api/miauth/{session}/check
```

在此：

- `{host}` 部分替换为用户服务器的主机名。
- `{session}` 部分替换为会话 ID。

响应中包含的属性如下：

| 名称      | 说明       |
| ------- | -------- |
| `token` | 用户的访问令牌。 |
| `user`  | 用户信息。    |

# 常见问题（FAQ）

这里收录了使用 Misskey 时的常见问题。关于 Misskey 项目本身的常见问题，请参见[此处](../../about-misskey#常见问题)。

## 有 Android 或 iOS 的App应用吗？

Misskey 项目并未开发面向此类移动操作系统的原生应用，但存在一些第三方应用。详情请参见[此处](./apps)。<br>

不过，由于第三方应用对 Misskey 最新功能的适配难免会有所滞后，若您没有特别偏好，建议使用 Misskey 官方的 Web 客户端 Misskey Web。另外，Misskey Web 支持 PWA，因此也可以像原生应用一样使用。详情请参见[此处](/docs/for-users/stepped-guides/how-to-use-pwa/)。

## 可以用面向 Mastodon 的应用登录吗？

由于 Misskey 与 Mastodon 的 API 不兼容，除少数例外情况外，无法通过第三方的 Mastodon 应用或 Web 客户端等来使用 Misskey。<br>
请使用 Misskey 官方的 Web 客户端 Misskey Web。

## misskey.io 是 Misskey 项目的官方服务器吗？

misskey.io 并非 Misskey 的官方服务器，也不属于 Misskey 项目本身。misskey.io 不设特定主题，定位为通用服务器，并且可以轻松注册，是 Misskey 规模最大的服务器。

## 与 MisskeyHQ（日本）股份有限公司是什么关系？

MisskeyHQ 股份有限公司是一家以运营 misskey.io 为目的的公司。因此，它与 Misskey 开发项目没有直接关系。
不过，作为 Misskey 项目负责人的 syuilo 也以公司高层身份参与其中，双方保持合作，共同推动 Misskey 的发展。

## 「Misskey」这个名字的由来是什么？

该名称取自主开发者 syuilo 在构思名称时偶然听到的 May'n 的歌曲《Brain Diver》歌词。

## 如何关注来自其他 Misskey、Pleroma 或 Mastodon 等服务器的用户？

从菜单中选择“搜索”，并按以下格式输入用户账号。用户账号由用户名与该用户所属服务器（或服务器主机名）组成。不仅是 Misskey，在 Mastodon、Pleroma 等许多分布式软件中，这种用户账号格式也很常见。<br>

用户账号格式：`@用户名@Misskey、Pleroma 或 Mastodon 服务器的主机名`<br>
实际例子：`@syuilo@misskey.io`<br>

这种用户账号格式并非适用于所有分布式软件，但使用该格式，您通常可以关注其他服务器上的用户（即使对方所属的服务器是由不同的分布式软件实现）。

## 如何取消转发？

点击显示的转发时间旁边的“...”，选择“取消转发”。关于转发的更多信息请参见[此处](../features/note/#renote)。

## 不希望显示帖子中的 URL 预览

Misskey 的专用标记语言 MFM（Markup language For Misskey）提供了用于禁用 URL 预览的语法。详情请参见 MFM 速查表。您可以在您所属的服务器中输入以下地址查看 MFM 速查表：`https://您的服务器域名/mfm-cheat-sheet`

## 我如何添加、编辑或删除自定义表情？

只有您所属服务器的管理员才能添加、编辑或删除自定义表情。如需进行这些操作，请直接联系服务器管理员。

## 我想要开发一个Bot机器人

您可以使用 Misskey API 开发Bot机器人。详情请参见[此处](../../for-developers/api/)。

## 帖子翻译功能使用了什么服务？

我们使用机器翻译服务 [DeepL 翻译](https://www.deepl.com/)。

## 我想咨询关于服务的问题

“Misskey”是用于搭建 Web 服务的软件，以及该项目的名称。因此，<b>“Misskey”本身并不是某个 Web 服务。</0><1>
此外，Misskey Project 不运营或管理任何 Web 服务，因此关于“使用 Misskey 搭建的 Web 服务”的咨询，请联系该 Web 服务自身的咨询渠道。<br>
需要注意的是，有些人会用“Misskey”来指代“基于 Misskey 搭建的 Web 服务”。

## 赞助金将如何使用？

将用于 Misskey 的长期开发与维护。

具体而言，主要用途是向全职开发者（目前为 @syuilo）支付薪资（报酬）。
Misskey Project 正是因为有全职开发者才能维持运作。
此外，也会用于向非全职开发者支付报酬，例如以业务委托方式合作时的报酬，以及不定期发放的补助金等。

此外，虽然占比不高，但在部分情况下也会用于以下开支：<br>1) 相关人员的接待交际费
2) 对其他开源软件（OSS）的支持费用
3) 域名维护费用
4) 开发所需的各类软件与服务使用费
5) 资料采购费用（例如书籍等）
6) 设备费用（例如用于运行验证的设备等）
7) 商标等权利的维护费用
8) 税务会计师服务费
9) 周边物品（例如 Misskey 卡片等）的相关费用（制作费、配送费等）
10) 素材（logo、图标、插画、音效等）的委托制作费用
11) 虚拟办公服务费
12) 等等

关于广告，目前并未实施，因此不会产生宣传广告费用。

另外，请注意：原则上，收到的赞助金在税务上会被视为收入，因此需要依法缴税；同时，收款方还需要向用于收款或转账的支付服务支付手续费。因此，赞助金并不能全部用于上述用途，敬请理解。

## 我已搭建服务器，是否需要向总务省提交电气通信事业备案（届出）？（日本专属问题）

### 简短回答

如果以**营利为目的**运营：**需要**。否则：**不需要**。

### 详细回答

根据[总务省文件的说明](https://www.soumu.go.jp/main_content/000477428.pdf)，需要提交备案的“经营者”的定义是：“通过提供服务并收取相应费用，从而获取收益的人”。因此，除非你以商业方式、以获取利润为目的来运营服务器，否则不需要提交备案。另外，即使通过捐赠或广告等方式获得了收入，只要这些收入仅用于支付服务器维护费用的范围内，也不属于需要提交备案的经营者。（已向总务省确认）<br>
更详细的信息，请参阅总务省的[《电气通信事业参入手册》](https://www.soumu.go.jp/main_content/000477428.pdf)等资料，或直接向总务省咨询。
此外，即使不需要备案，也可以自愿进行备案。

## 是否因为具备私信（DM）功能，就需要提交电气通信事业备案（届出）？

结论与[“我已搭建服务器，是否需要向总务省提交电气通信事业备案（届出）？”](#我已搭建服务器是否需要向总务省提交电气通信事业备案届出日本专属问题)一节中的说明相同。
无论所运营的服务具备何种功能，只要不属于“通过收取费用以获取收益”的情形，就不会被视为需要提交备案的经营者。

## 公开名称中包含“Misskey”的服务可以吗？

“Misskey”是 Misskey Project 在日本的注册商标，但公开名称中包含“Misskey”（例如 Misskey Tools 等）的服务并不会有问题。
我们不打算在这种情况下收取授权费。

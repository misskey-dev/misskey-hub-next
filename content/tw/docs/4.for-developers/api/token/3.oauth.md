# 透過 OAuth 方式取得存取權杖

若要為使用應用程式的使用者（以下簡稱「使用者」）取得存取權杖，請依照下列步驟請求核發。

:::tip

以下說明的方法稱為 [OAuth 2.0](https://datatracker.ietf.org/doc/html/rfc6749.html)。一般的 OAuth 需要建立應用程式，但透過 [IndieAuth](https://indieauth.spec.indieweb.org/) 的擴充功能，現在無須事先建立應用程式即可使用。

由於支援 OAuth 方式的函式庫相當多，建議盡可能使用現成的函式庫。

目前，必須要有網頁才能使用此方式。如果無論如何都無法準備網頁，或者想要支援 Misskey 2023.9.0 以前的版本，請使用下列方式：

- [透過 Misskey 專用的 MiAuth 方式取得存取權杖](./miauth.md)
- [透過建立應用程式取得存取權杖（舊方法）](./app.md)

:::

## 步驟1

建立一個用於介紹應用程式的網頁。請確保該網頁可透過 HTTPS 網址存取。請在網頁的某處編寫如下的 HTML 程式碼：

```html
<!-- （必填項目）href 的位址將會作為授權碼的重新導向目的地（轉送目標）。 -->
<link rel="redirect_uri" href="/redirect">

<div class="h-app">
  <!-- （選填 / v2025.3.0 起）可指定應用程式的標誌（Logo）。建議使用長寬比為 1:1 的標誌。 -->
  <img src="/logo.png" class="u-logo">
  <!-- （選填）指定要顯示給使用者看的應用程式名稱。若未指定，則此頁面的網址將會成為預設名稱。 -->
  <!-- （選填）href 屬性必須設定為連結至此頁面（＝此應用程式介紹頁面的 URL）。 -->
	<a href="/" class="u-url p-name">My Misskey App</a>
</div>
```

稍後，授權碼（Authorization Code）將會被轉送至 redirect_uri 所指定的位址。

## 步驟2

產生 PKCE 的 `code_verifier` 與 `code_challenge` 字串，以及 `state` 字串。

- `code_verifier`：長度限制為最少 43 個字元，最多 128 個字元，且僅限使用英文字母大小寫與 -._~ 中的字元。
- `code_challenge`：將 `code_verifier` 字串透過 SHA256 演算法進行雜湊處理後，再以 base64url 編碼的結果。
- `state`：沒有特別的限制，請使用隨機字串。

:::danger

這些字串請每次都重新產生，切勿重複使用。

:::

:::tip

建議使用像是 [pkce-challenge](https://www.npmjs.com/package/pkce-challenge) 這類的函式庫，或是直接使用 OAuth 函式庫中內建的 PKCE 功能。

:::

:::tip{label='例'}

```js
import crypto from "node:crypto";

const chars = "ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz-._~";
const codeVerifier = new Array(128)
  .fill(0)
  .map(() => chars[Math.floor(chars.length * Math.random())])
  .join("");
console.log('code_verifier', codeVerifier);

const codeChallenge = crypto
  .createHash("sha256")
  .update(codeVerifier, "ascii")
  .digest("base64url");
console.log('code_challenge', codeChallenge);

const state = crypto.randomUUID();
console.log('state', state);
```

:::

## 步驟3

取得目標伺服器的 OAuth 資訊。資料為 JSON 格式。

```
https://{host}/.well-known/oauth-authorization-server
```

將 `{host}` 替換為使用者伺服器的主機。主機通常由使用者輸入。

在這裡，我們會使用到 `authorization_endpoint` 與 `token_endpoint`。

:::tip

在下一個步驟中會用到的 `scope` 資訊，也可以透過 `scopes_supported` 欄位來確認。

:::

## 步驟4

在使用者的瀏覽器中顯示應用程式授權表單。您可以透過以下格式的 URL 開啟授權表單：

```
{authorization_endpoint}?client_id={client_id}&response_type=code&redirect_uri={redirect_uri}&scope={scope}&code_challenge={code_challenge}&code_challenge_method=S256&state={state}
```

在此：

- `{authorization_endpoint}`：替換為在前一個步驟取得的位址。
- `{client_id}`：替換為應用程式介紹頁面的位址。
- `{code_challenge}`：替換為之前產生的 `code_challenge` 字串。
- `code_challenge_method`：請固定設為 `S256`。
- `{redirect_uri}`：替換為在介紹頁面中設定的轉送目的地（重新導向）位址。
- `{scope}`：替換為應用程式要求取得的權限。請將要求的權限以空格` `分隔列出。可以在[此處](../permission.md)查看權限清單。
- `{state}`：替換為之前產生的 `state` 字串。

:::tip{label='例'}

```
https://misskey.local/oauth/authorize?client_id=http%3A%2F%2Fexample.com&code_challenge=C6hwMO2bmIzg3nqppTE9b79fvuOjlrKmH2xNiZSMHzw&code_challenge_method=S256&response_type=code&redirect_uri=http%3A%2F%2Fexample.com%2Fredirect&scope=write%3Anotes&state=87c11f05-86eb-4eb2-9057-f6a98fc5e9ab
```

:::

## 步驟5

當使用者允許應用程式存取後，授權碼會以 URL 參數的形式被轉送至 `redirect_uri` 的位址。

| 名稱      | 說明                    |
| ------- | --------------------- |
| `code`  | 使用者的授權碼。              |
| `state` | 授權請求時所使用的 `state` 字串。 |

:::tip{label='例'}

```
https://example.com/redirect?code=...&state=87c11f05-86eb-4eb2-9057-f6a98fc5e9ab
```

:::

請確認收到的 `state` 字串是否與當初發送的一致，確認無誤後再進入下一個步驟。

## 步驟6

使用轉送過來的授權碼，透過 POST 方法請求 Access Token。請求的目的地為 `token_endpoint`。資料格式可支援 `application/json` 與 `application/x-www-form-urlencoded`。各項參數設定如下：

| 名稱              | 說明                           |
| --------------- | ---------------------------- |
| `grant_type`    | 始終設為 `authorization_code`。   |
| `client_id`     | 授權請求時所使用的 `client_id` 字串。    |
| `redirect_uri`  | 授權請求時所使用的 `redirect_uri` 字串。 |
| `scope`         | 授權請求時所使用的 `client_id` 字串。    |
| `code`          | 取得的授權碼。                      |
| `code_verifier` | 之前產生的 `code_verifier` 字串。    |

:::tip{label='例'}

```js
const res = await fetch(endpoint, {
  method: "POST",
  body: JSON.stringify({
    grant_type: "authorization_code",
    client_id: "https://example.com",
    redirect_uri: "https://example.com/redirect",
    scope: "write:notes",
    code: "...",
    code_verifier: "hjjbCYDmDpSLjirkO-PrfWKsRhDdJr-PAEGRClRwzUKlmFIIIrZNmSvUIraeIa~WqbqQnfbJV-Hc_IfuQkesBYUpukUi~lInDfU_AZjoZqbU.ioQTRzaFfZFfGnT-OAA",
  }),
  headers: {
    "Content-Type": "application/json"
  }
});
```

:::

回應內容將會是 JSON 物件格式，您可以從該物件中取得並使用 `access_token`。

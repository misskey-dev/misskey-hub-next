# Streaming API

:::tip

建议您先阅读 [Misskey API](../index.md) 文档。

:::

使用 Streaming API，您可以实时接收各种信息（例如时间轴上的新帖子、被表情回应、被关注等）或执行各种操作。

## 连接到流 (Stream)

要使用 Streaming API，首先必须通过 **WebSocket** 连接到 Misskey 服务器。

请通过 WebSocket 连接到以下格式的 URL：

```
wss://{host}/streaming?i={token}
```

在此：

- `{host}` 部分请替换为您想连接的服务器主机名。
- `{token}` 部分请替换为用户的访问令牌 (Access Token)。

:::tip

访问令牌可以省略，但在这种情况下将被视为未登录状态使用，可接收的信息和可执行的操作将受到限制。

:::

连接到流后，虽然可以进行后述的“帖子捕捉 (Note Capture)”，但在此阶段尚无法接收例如时间轴新帖子等信息。
要接收这类事件，必须连接到流上的频道 (Channel)（详见后述）。

**流上的所有交互皆以 JSON 格式进行。**

## 频道 (Channels)

Misskey 的 Streaming API 有“频道”的概念。这是用来分离发送与接收信息的机制。通过在流上连接到频道，便能接收各种信息或发送信息。

:::tip

在同一个流连接中，可以同时连接到多个频道。

:::

以下将说明频道的使用方法。关于有哪些可用的频道，请参阅[频道列表](./channel/index.md)。

### 连接到频道

要连接到频道，请将以下数据以 JSON 格式发送到流：

```js
{
	type: 'connect',
	body: {
		channel: 'xxxxxxxx',
		id: 'foobar',
		params: {
			...
		}
	}
}
```

在此：

- `channel`：设置想连接的频道名称。请参阅[频道列表](./channel/index.md)。
- `id`：设置一个任意 ID 以便与该频道交互。由于流中会流动各种消息，因此需要识别消息是来自哪个频道的。此 ID 可以是 UUID 或随机数。
- `params`：连接到频道时所需的参数。不同频道在连接时所需的参数各异。连接到不需要参数的频道时，此属性可省略。

:::tip

ID 并非针对每个“频道种类”，而是针对每次“频道连接”。因为可能会发生使用不同参数多次连接到同一频道的情况。

:::

### 接收来自频道的消息

例如时间轴频道，当有新帖子时会发出消息。通过接收该消息，便能实时得知时间轴有了新帖子。

当频道发出消息时，会有如下的数据以 JSON 格式流入流：

```js
{
	type: 'channel',
	body: {
		id: 'foobar',
		type: 'something',
		body: {
			some: 'thing'
		}
	}
}
```

在此：

- `id`：会设置为前述连接到该频道时所设置的 ID。藉此便能得知此消息是来自哪个频道。
- `type`：设置消息的种类。根据频道不同，流入的消息种类也会有所不同。
- `body`：设置消息的内容。根据频道不同，流入的消息内容也会有所不同。

### 向频道发送消息

某些频道不仅接收消息，还允许我们发送消息以执行某些操作。

要向频道发送消息，请将以下数据以 JSON 格式发送到流：

```js
{
	type: 'channel',
	body: {
		id: 'foobar',
		type: 'something',
		body: {
			some: 'thing'
		}
	}
}
```

在此：

- `id`：设置前述连接到该频道时所设置的 ID。藉此让系统识别此消息是发送给哪个频道的。
- `type`：设置消息的种类。不同频道所接受的消息种类各异。
- `body`：设置消息的内容。不同频道所接受的消息内容各异。

### 断开频道连接

要断开与频道的连接，请将以下数据以 JSON 格式发送到流：

```js
{
	type: 'disconnect',
	body: {
		id: 'foobar'
	}
}
```

在此：

- `id`：设置前述连接到该频道时所设置的 ID。

## 帖子捕捉 (Note Capture)

Misskey 提供了称为“帖子捕捉”的机制。这是通过流接收指定帖子事件的功能。

例如，假设您获取了时间轴并显示给用户。此时若有人对该时间轴中的某则帖子进行了表情回应。
然而，客户端无从得知某则帖子被按了表情回应，因此无法实时将表情回应反映并显示在时间轴的帖子上。

为了解决这个问题，Misskey 准备了帖子捕捉机制。一旦捕捉了帖子，就能接收关于该帖子的事件，因此可以实时反映表情回应等变化。

以下将说明帖子捕捉功能的使用方法。关于有哪些捕捉事件，请参阅[捕捉事件列表](./note-capture-events.md)。

### 捕捉帖子

要捕捉帖子，请向流发送如下消息：

```js
{
	type: 'subNote',
	body: {
		id: 'xxxxxxxxxxxxxxxx'
	}
}
```

在此：

- `id`：设置想捕捉的帖子 ID。

发送此消息后，即代表向 Misskey 请求了捕捉，此后关于该帖子的事件便会流入。

例如，若该帖子被按了表情回应，将会流入如下消息：

```js
{
	type: 'noteUpdated',
	body: {
		id: 'xxxxxxxxxxxxxxxx',
		type: 'reacted',
		body: {
			reaction: 'like',
			userId: 'yyyyyyyyyyyyyyyy'
		}
	}
}
```

在此：

- `body` 内的 `id`：设置发生事件的帖子 ID。
- `body` 内的 `type`：设置事件的种类。
- `body` 内的 `body`：设置事件的详细内容。

### 解除帖子捕捉

当该帖子不再显示于画面上，不再需要接收关于该帖子的事件时，请申请解除捕捉。

发送以下消息：

```js
{
	type: 'unsubNote',
	body: {
		id: 'xxxxxxxxxxxxxxxx'
	}
}
```

在此：

- `id`：设置想解除捕捉的帖子 ID。

发送此消息后，此后便不再流入关于该帖子的事件。

## 2025.12.2

釋出時間: 2025/12/22

### Note

針對 v2025.12.0 中進行的「將 config 的 `trustProxy` 預設值變更為 `false`」一事，為了緩解如果沒有根據環境正確設定將會導致難以登入的狀況，我們進行了以下應對。

**如果沒有正確設定，可能會導致上述的錯誤，或者增加安全風險。請務必確認目前的 config，並在必要時更改數值。**

- 關於 `trustProxy`，在預設情況下（config 中未設定數值的狀態），現在會信任回環位址和本地 IP 位址空間。
- 已更詳細地描述了 `trustProxy` 的設定方法。
- 如果在反向代理或 CDN 等更上層想要設定速率限制，或是作為緊急時的暫時緩解措施，現在可以停用 Misskey 內部基於 IP 位址的速率限制。

### General

- 相依套件更版

### Client

- Enhance: 新增 Deck 的 UI 說明
- Enhance: 新增選項以防止瀏覽器清除設定
- Fix: 修復沒有版本標記的 Play 無法正確運作的問題  
  沒有版本標記的將作為 v0.x 系統執行。如果想在 v1.x 系統中運作，請務必包含版本標記。
- Fix: 修復在 Deck UI 中將選單位置設為下方時，不會顯示刪除個人資料按鈕的問題
- Fix: 修復部分 Unicode 表情符號的反應不會變成按鈕的問題

### Server

- Enhance: 現在可以停用 Misskey 內部基於 IP 位址的速率限制
  - 這預計用於在反向代理或 CDN 等其他層級額外設定速率限制，或是作為本地測試用途。
  - 預設為 `enableIpRateLimit: true`（Misskey 內部基於 IP 位址的速率限制為有效）。
- Fix: 改善控制面板的工作佇列頁面中所使用的部分 API 回應速度

## 2025.8.0

釋出時間: 2025/08/31

### Note

- 支援的 Node.js 最低版本變為 **22.15.0**

### General

- 刪除貼文時，不再同時刪除相關聯的貼文。
  - 在 API 中，將會出現以往沒有的情況，例如「存在 `replyId` 但 `reply` 為 `null`」或「存在 `renoteId` 但 `renote` 為 `null`」。
- 實作了定期刪除舊有遠端貼文的功能。
  - 可從控制面板→效能→Remote Notes Cleaning 啟用。
  - 可以防止資料庫過度膨脹。
  - 如果現有伺服器啟用此功能，因處理量大，儲存空間使用量可能會暫時增加。
    - 要抑制增加量，請將最大處理持續時間設定得比預設短。
  - 如果對資料庫大小沒有效果，請確認是否啟用了 `autovacuum`。
- 在伺服器初始設定完成前，聯邦功能將不會開啟。
- 日文中的公開範圍名稱「ダイレクト (Direct)」已改名為「指名」。
  - 名稱更符合實際運作，不熟悉的人也更容易理解。
  - 在其他服務中相當於「直接訊息 (DM)」的 Misskey 功能是「聊天 (Chat)」（在舊版的 Misskey 中，該功能也是「直接訊息」而不是「聊天」），但存在名為「直接發文」的功能會讓人誤以為那是直接訊息功能。
    - 未來，「聊天」的名稱可能會改回「直接訊息」。
- 更新了 mfm.js
  - Enhance: 支援 Unicode 15.1 及 16.0 中收錄的表情符號
  - Enhance: 支援 `acct` 中帶有 `.` 的使用者標記

- Enhance: 使用者搜尋可透過身分組政策進行限制

  - 擴充功能需要對應 1.x 版本。
  - Play 可直接運作，但新建的預設會是 1.x。
  - 從以前版本被停用的 `note_view_interruptor` 現已啟用。
    - 處理程序必須是同步的。
- Feat: 安全模式。
  - 當使用擴充功能、佈景主題、自訂 CSS 導致用戶端啟動發生問題時，可以停用這些項目來啟動。
  - 可透過以下方法啟動安全模式：
    - 連按 `g` 鍵
    - 在 URL 加上 `?safemode=true`
    - 在 PWA 的捷徑中選擇 Safemode 啟動
- Feat: 可以選擇未登入時顯示的首頁風格
  - 控制面板→品牌化→入口頁面風格
- Feat: 頁面的分頁列可以顯示在底部
- Feat: (實驗性) 在 iOS 上可啟用觸覺回饋
- Feat: 可以搜尋控制面板
- Enhance: 「自動顯示更多」選項已啟用，並提升了穩定性
- Enhance: 支援土耳其語 (tr-TR)
- Enhance: 不再載入不必要的翻譯資料，提升效能
- Enhance: 圖片特效參數名稱的多語言支援
- Enhance: 現在可以以 1 個月為單位自由指定隱藏貼文的相對期間

### Changes

#### For server admins

- 需要 Node.js 18.x 或更高版本
- 需要 PostgreSQL 15.x。
  - Misskey 在 13.0.0 尚未使用 15 的專屬功能，但未來可能會使用。
  - 在 Docker 環境中更新 PostgreSQL 的指南請見此: https://github.com/misskey-dev/misskey/pull/9641#issue-1536336620
- 已移除對 Elasticsearch 的支援。
  - 取而代之的是，我們正在構想未來可設定任意搜尋提供者的機制。若使用該機制，未來依然能照常使用 Elasticsearch。
- 已從 Yarn 遷移至 pnpm。
  建議啟用 corepack: `sudo corepack enable`
- 站點封鎖現在也會套用於子網域
- 伴隨身分組的導入，一些功能已與身分組整合：
  - 審查員已與身分組整合。過去的審查員資訊將會遺失，因此請預先記錄審查員名單，並在更新後建立審查員身分組重新分配。
  - 靜默功能已與身分組整合。過去被靜默的使用者將被特赦，因此建議預先記錄靜默名單。
  - 個別使用者的雲端硬碟容量設定已與身分組整合。
  - 站點預設的雲端硬碟容量設定已與身分組整合。更新後，請在基礎身分組或條件式身分組中編輯雲端硬碟容量。
  - LTL/GTL 的開放狀態已與身分組整合。
- 不再以 root 身分執行 Docker。若使用 Docker 且未開啟物件儲存體，請執行 `chown -hR 991.991 ./files`。  
  https://github.com/misskey-dev/misskey/pull/9560

#### For users

- 已移除貼文的關注功能
- 當投票被勾選時，將不再產生通知
- 已移除貼文的數學公式嵌入功能
- 已無法再建立新的動態 Pages。
  - 取而代之的是，實作了可使用 AiScript 建立更具彈性動態內容的 Misskey Play 功能。
- AiScript 已更新至 0.12.2。
  - 關於 0.12.x 的變更點請見此 https://github.com/syuilo/aiscript/blob/master/CHANGELOG.md#0120。
  - 低於 0.12.x 的外掛程式將無法讀取。
- 不再支援 iOS 15 及更低版本的裝置
- 不再支援 Firefox 110 及更低版本。
  - 在 109 版本中，只要啟用 ContainerQueries 標記即可正常使用。

#### For app developers

- API: `meta` 回應中不再包含 `emojis` 屬性。
  - 若要取得自訂表情符號列表資訊，請向 `emojis` 端點發送請求。

- API: `user` 實體中不再包含 `avatarColor` 及 `bannerColor` 屬性

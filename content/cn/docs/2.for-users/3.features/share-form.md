# 分享表单

将 Misskey Web 的根路径设为 `/share`，即可打开用于分享的发帖页面。使用该分享表单时，对于希望别的用户在 Misskey 上分享自己网站内容的外部网站站主来说，会非常方便。

可以用 URL 的查询参数来指定分享内容等多个选项。

## 查询参数

:::tip

所有参数均为**可选**，并非必填。

:::

| 名称      | 说明                                                                                 |
| ------- | ---------------------------------------------------------------------------------- |
| `title` | 标题。会以「\[ 标题内容 \]」的形式插入到正文开头。 |
| `text`  | 正文。                                                                                |
| `url`   | URL。将被插入到正文末尾。                                                                     |

### 回复信息

指定以下任意一项后，即可将该帖子设为对指定笔记的回复。

| 名称         | 说明                    |
| ---------- | --------------------- |
| `replyId`  | 要回复的帖子 ID。            |
| `replyUri` | 要回复的帖子网址（用于远程服务器的帖子）。 |

### 转发信息

指定以下任意一项后，即可将该帖子设为对指定笔记的转发（引用）。

| 名称          | 说明                    |
| ----------- | --------------------- |
| `renoteId`  | 要转发的帖子 ID。            |
| `renoteUri` | 要转发的帖子网址（用于远程服务器的帖子）。 |

### 可见性

可以通过以下选项指定可见性。

| 名称               | 说明                                                                            |
| ---------------- | ----------------------------------------------------------------------------- |
| `visibility`     | `public` （公开）， `home` （首页）， `followers` （仅关注者）， `specified` （指定用户）中选一个        |
| `localOnly`      | 不参与联合（仅限本地），使用 0(false) 关闭，或 1(true) 开启 |
| `visibleUserIds` | 所指定的用户的 ID（以逗号分隔）                                                             |
| `visibleAccts`   | 所指定的用户的 [acct](../resources/glossary/#acct)(逗号分隔)          |

:::warning

如果将 `visibility` 指定为 `specified`，则也必须指定 `visibleUserIds` 或 `visibleAccts`。

:::

### 附件

可以通过以下选项指定附件。

| 名称        | 说明               |
| --------- | ---------------- |
| `fileIds` | 要附加的文件 ID（以逗号分隔） |

### 基本参数

原则上可以直接使用上述介绍的参数，但\*\*依赖于各个服务器的参数（例如用户 ID、文件 ID 等）无法使用。\*\*如果指定了，则会在 Misskey Hub 上被删除。

### 独有功能

#### 推荐服务器功能

通过在 URL 参数 `manualInstance` 中指定 Misskey 服务器的域名，可以以“来自分享源网站的推荐”的形式，单独显示指向该服务器的链接。例如在引导用户前往您自己的服务器时，可以使用此功能。

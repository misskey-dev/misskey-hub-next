# Streaming API

:::tip

建議您先閱讀 Misskey API 文件。

:::

使用 Streaming API，您可以即時接收各種資訊（例如時間軸上的新貼文、被表情反應、被追蹤等）或執行各種操作。

## 連線至串流 (Stream)

要使用 Streaming API，首先必須透過 **WebSocket** 連線至 Misskey 伺服器。

請以 WebSocket 連線至以下格式的 URL：

```
wss://{host}/streaming?i={token}
```

在此：

- `{host}` 的部分，請替換為您想連線的伺服器主機名稱。
- `{token}` 的部分，請替換為使用者的存取權杖 (Access Token)。

:::tip

存取權杖可以省略，但在這種情況下將視為未登入狀態使用，可接收的資訊和可執行的操作將受到限制。

:::

連線至串流後，雖然可以進行後述的「貼文捕捉」，但在此階段尚無法接收例如時間軸新貼文等資訊。
要接收這類事件，必須連線至串流上的頻道 (Channel)（詳見後述）。

**串流上的所有互動皆以 JSON 格式進行。**

## 頻道 (Channels)

Misskey 的 Streaming API 有「頻道」的概念。這是用來分離傳送與接收資訊的機制。
透過在串流上連線至頻道，便能接收各種資訊或發送資訊。

:::tip

在同一個串流連線中，可以同時連線至多個頻道。

:::

以下將說明頻道的使用方法。關於有哪些可用的頻道，請參閱[頻道列表](./channel/index.md)。

### 連線至頻道

要連線至頻道，請將以下數據以 JSON 格式發送至串流：

```js
{
	type: 'connect',
	body: {
		channel: 'xxxxxxxx',
		id: 'foobar',
		params: {
			...
		}
	}
}
```

在此：

- `channel`：設定想連線的頻道名稱。請參閱[頻道列表](./channel/index.md)。
- `id`：設定一個任意 ID 以便與該頻道互動。由於串流中會流動各種訊息，因此需要識別訊息是來自哪個頻道的。此 ID 可以是 UUID 或隨機數。
- `params`：連線至頻道時所需的參數。不同頻道在連線時所需的參數各異。連線至不需要參數的頻道時，此屬性可省略。

:::tip

ID 並非針對每個「頻道種類」，而是針對每次「頻道連線」。因為可能會發生使用不同參數多次連線至同一頻道的情況。

:::

### 接收來自頻道的訊息

例如時間軸頻道，當有新貼文時會發出訊息。透過接收該訊息，便能即時得知時間軸有了新貼文。

當頻道發出訊息時，會有如下的數據以 JSON 格式流入串流：

```js
{
	type: 'channel',
	body: {
		id: 'foobar',
		type: 'something',
		body: {
			some: 'thing'
		}
	}
}
```

在此：

- `id`：會設定為前述連線至該頻道時所設定的 ID。藉此便能得知此訊息是來自哪個頻道。
- `type`：設定訊息的種類。根據頻道不同，流入的訊息種類也會有所不同。
- `body`：設定訊息的內容。根據頻道不同，流入的訊息內容也會有所不同。

### 向頻道發送訊息

某些頻道不僅接收訊息，還允許我們發送訊息以執行某些操作。

要向頻道發送訊息，請將以下數據以 JSON 格式發送至串流：

```js
{
	type: 'channel',
	body: {
		id: 'foobar',
		type: 'something',
		body: {
			some: 'thing'
		}
	}
}
```

在此：

- `id`：設定前述連線至該頻道時所設定的 ID。藉此讓系統識別此訊息是發送給哪個頻道的。
- `type`：設定訊息的種類。不同頻道所接受的訊息種類各異。
- `body`：設定訊息的內容。不同頻道所接受的訊息內容各異。

### 斷開頻道連線

要斷開與頻道的連線，請將以下數據以 JSON 格式發送至串流：

```js
{
	type: 'disconnect',
	body: {
		id: 'foobar'
	}
}
```

在此：

- `id`：設定前述連線至該頻道時所設定的 ID。

## 貼文捕捉 (Note Capture)

Misskey 提供了稱為「貼文捕捉」的機制。這是透過串流接收指定貼文事件的功能。

例如，假設您獲取了時間軸並顯示給使用者。此時若有人對該時間軸中的某則貼文進行了表情反應。
然而，客戶端無從得知某則貼文被按了表情反應，因此無法即時將表情反應反映並顯示在時間軸的貼文上。

為了解決這個問題，Misskey 準備了貼文捕捉機制。一旦捕捉了貼文，就能接收關於該貼文的事件，因此可以即時反映表情反應等變化。

以下將說明貼文捕捉功能的使用方法。關於有哪些捕捉事件，請參閱[捕捉事件列表](./note-capture-events.md)。

### 捕捉貼文

要捕捉貼文，請向串流發送如下訊息：

```js
{
	type: 'subNote',
	body: {
		id: 'xxxxxxxxxxxxxxxx'
	}
}
```

在此：

- `id`：設定想捕捉的貼文 ID。

發送此訊息後，即代表向 Misskey 請求了捕捉，此後關於該貼文的事件便會流入。

例如，若該貼文被按了表情反應，將會流入如下訊息：

```js
{
	type: 'noteUpdated',
	body: {
		id: 'xxxxxxxxxxxxxxxx',
		type: 'reacted',
		body: {
			reaction: 'like',
			userId: 'yyyyyyyyyyyyyyyy'
		}
	}
}
```

在此：

- `body` 內的 `id`：設定發生事件的貼文 ID。
- `body` 內的 `type`：設定事件的種類。
- `body` 內的 `body`：設定事件的詳細內容。

### 解除貼文捕捉

當該貼文不再顯示於畫面上，不再需要接收關於該貼文的事件時，請申請解除捕捉。

發送以下訊息：

```js
{
	type: 'unsubNote',
	body: {
		id: 'xxxxxxxxxxxxxxxx'
	}
}
```

在此：

- `id`：設定想解除捕捉的貼文 ID。

發送此訊息後，此後便不再流入關於該貼文的事件。

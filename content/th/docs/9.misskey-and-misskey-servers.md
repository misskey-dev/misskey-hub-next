# Misskey กับเซิร์ฟเวอร์ Misskey

‘Misskey’ กับ ‘เซิร์ฟเวอร์ Misskey’ เป็นคนละสิ่งกัน

อย่างไรก็ตาม เนื่องจากชื่อเรียกมีความคล้ายกัน <b> อีกทั้งคำว่า “เซิร์ฟเวอร์ Misskey” มักถูกย่อและเรียกเพียงว่า “Misskey” อยู่บ่อยครั้ง จึงทำให้เกิดความสับสนขึ้นได้ง่ายมาก</b>

ในความเป็นจริง มีการติดต่อสอบถามจำนวนมากที่เข้าใจผิด โดยรับรู้ “เซิร์ฟเวอร์ Misskey” ว่าเป็น “Misskey”
(Misskey ไม่ได้เป็นผู้ดำเนินการเซิร์ฟเวอร์ Misskeyและเว็บไซต์นี้เป็นเว็บไซต์อย่างเป็นทางการของ “Misskey” ไม่ใช่เว็บไซต์ของ “เซิร์ฟเวอร์ Misskey”)

โปรดตรวจสอบอีกครั้งว่าไม่ได้สับสนระหว่างสองสิ่งนี้อยู่

## 「Misskey」

Misskey คือ <b>ซอฟต์แวร์ฟรีและโอเพนซอร์ซสำหรับสร้างบริการโซเชียลเน็ตเวิร์ก (SNS)</b> ที่พัฒนาโดยชุมชนอาสาสมัครที่มีชื่อว่า “Misskey Project”

คำว่า โอเพนซอร์ส หมายถึง ซอร์สโค้ด (แบบแปลน) ของซอฟต์แวร์ถูกเปิดเผยต่อสาธารณะ คำว่า ฟรี หมายถึง การใช้งานซอร์สโค้ดไม่เสียค่าใช้จ่าย และทุกคนสามารถนำไปใช้ได้อย่างอิสระ

บางครั้ง Misskey ถูกแนะนำว่าเป็น “SNS” แต่ความเข้าใจดังกล่าวไม่ถูกต้อง

Misskey เป็นเพียง “ซอฟต์แวร์สำหรับสร้างเว็บบริการ” ไม่ใช่ ”ตัวเว็บบริการ” ที่มีการสมัครสมาชิกหรือเข้าสู่ระบบ

:::tip

ดังนั้น จึงไม่มีสิ่งที่เรียกว่า “ผู้ดูแลระบบ Misskey”

หากเคยมีประสบการณ์ในการสมัครสมาชิกหรือเข้าสู่ระบบ สิ่งนั้นไม่ใช่ Misskey แต่เป็น เซิร์ฟเวอร์ Misskey

:::

:::warning

อาจมีบัญชีสแปมที่แอบอ้างใช้ชื่อว่า “ผู้ดูแลระบบ Misskey” หรือ “ฝ่ายสนับสนุน Misskey” เป็นต้น

ดังที่กล่าวไว้ องค์กรหรือช่องทางติดต่อในลักษณะดังกล่าว ไม่มีอยู่จริงตั้งแต่แรกโปรดใช้ความระมัดระวัง

:::

เนื่องจากซอร์สโค้ดของ Misskey เปิดให้ทุกคนสามารถนำไปใช้ได้อย่างอิสระ จึงมี SNS ที่สร้างขึ้นโดยใช้ Misskey (กลายเป็นเซิร์ฟเวอร์ Misskey) ถูกเผยแพร่บนอินเทอร์เน็ตโดยผู้คนหลากหลายกลุ่ม

นอกจากนี้ ทุกคนยังสามารถเข้าร่วมในการพัฒนาได้

## 「เซิร์ฟเวอร์ Misskey」

เซิร์ฟเวอร์ Misskey หมายถึง <b>“SNS ที่ถูกสร้างขึ้นโดยใช้ Misskey”</b>

เช่น SNS อย่าง “misskey.io”, “misskey.design”, “nijimiss.moe” ล้วนเป็น เซิร์ฟเวอร์ Misskey (หรือ เซิร์ฟเวอร์สาย Misskey)

ทั้งนี้ เซิร์ฟเวอร์ Misskey ไม่ได้ประกอบด้วย Misskey เพียงอย่างเดียว แต่ถูกสร้างขึ้นโดยการผสานซอฟต์แวร์หลายชนิดเข้าด้วยกัน เช่น ซอฟต์แวร์ฐานข้อมูล PostgreSQL, Redis และซอฟต์แวร์เว็บเซิร์ฟเวอร์อย่าง Nginx เป็นต้น

ณ ปี 2025 มีการยืนยันว่า มีเซิร์ฟเวอร์ Misskey มากกว่า 1,000 เซิร์ฟเวอร์บนอินเทอร์เน็ต

เซิร์ฟเวอร์เหล่านี้อาจถูกดูแลโดยบุคคลทั่วไป หรือองค์กร/นิติบุคคล ก็ได้

:::tip

ผู้ดูแล นโยบายการดำเนินงาน และข้อกำหนดการใช้งานจะแตกต่างกันไปในแต่ละเซิร์ฟเวอร์ Misskey

Misskey Project (ชุมชนผู้พัฒนา Misskey) ไม่ได้เป็นผู้ดูแลหรือให้บริการเซิร์ฟเวอร์ Misskey และไม่ได้อยู่ในสถานะที่จัดการ ควบคุม หรือมีส่วนเกี่ยวข้องกับการดำเนินงานของเซิร์ฟเวอร์ Misskey แต่ละแห่ง

นอกจากนี้ ฟีเจอร์ต่างๆ เช่น เอโมจิที่กำหนดเอง, บทบาทของผู้ใช้, โฆษณา, หรือ รายการคำต้องห้าม(NG) ไม่ได้ถูกกำหนดมาโดยตัว Misskey เอง แต่เป็นสิ่งที่ผู้ดูแลของแต่ละเซิร์ฟเวอร์ตั้งค่าแยกกันไป

:::

ดังที่กล่าวไป เนื่องด้วย เซิร์ฟเวอร์ Misskey (รวมถึงสาย Fork) มักถูกเรียกรวมสั้นๆ ว่า “Misskey” การกล่าวถึง “Misskey” ในหลายกรณีอาจหมายถึงเซิร์ฟเวอร์ Misskey ใดเซิร์ฟเวอร์หนึ่งโดยเฉพาะ ไม่ใช่ตัวซอฟต์แวร์ Misskey เอง โปรดระมัดระวังจุดนี้ด้วย

นอกจากนี้ เพื่อสื่อว่าเป็นเซิร์ฟเวอร์ที่สร้างขึ้นโดยใช้ Misskey บนเซิร์ฟเวอร์นั้นอาจมีการแสดงโลโก้หรือไอคอนของ Misskey หรือมีข้อความอย่างเช่น “Powered by Misskey” ปรากฏอยู่

### 「เซิร์ฟเวอร์สาย Misskey」

เซิร์ฟเวอร์ที่ถูกสร้างขึ้นโดยใช้ซอฟต์แวร์ที่พัฒนามาจาก Misskey บางครั้งจะถูกรวมเรียกว่า เซิร์ฟเวอร์สาย Misskey

เซิร์ฟเวอร์สาย Misskey บางครั้งก็ถูกเรียกสั้นๆ ว่า เซิร์ฟเวอร์ Misskey เช่นกัน

เมื่อ SNS เรียกตนเองว่าเป็น “เซิร์ฟเวอร์ Misskey” ก็มีความเป็นไปได้ว่า SNS นั้นอาจกำลังไม่ได้ใช้งานซอฟต์แวร์ Misskey ต้นฉบับโดยตรง อาจจะเป็นรุ่นที่ fork ออกมา

## เช่น

### Webไซต์

Misskey ถ้าเปรียบเทียบ ก็เหมือนกับ “WordPress” ซึ่งเป็นซอฟต์แวร์สำหรับใช้สร้าง Webไซต์

ส่วนเซิร์ฟเวอร์ Misskey เทียบได้กับWebไซต์แต่ละแห่งที่ถูกสร้างขึ้นมาจริงๆ โดยใช้ “WordPress”

### อพาร์ตเมนต์

Misskey เปรียบเสมือน “แบบแปลนอพาร์ตเมนต์” ส่วนเซิร์ฟเวอร์ Misskey คือ “อพาร์ตเมนต์แต่ละหลังที่ถูกสร้างขึ้นจริงโดยอ้างอิงจากแบบแปลนนั้น”

Misskey Project เปรียบได้กับ “สำนักงานออกแบบ” ที่พัฒนาแบบแปลน ขณะที่ผู้ใช้เซิร์ฟเวอร์ Misskey ก็เทียบได้กับผู้อยู่อาศัยในอพาร์ตเมนต์

เช่นเดียวกับที่อพาร์ตเมนต์แต่ละแห่งมีเจ้าของหรือบริษัทจัดการแตกต่างกันไป เซิร์ฟเวอร์ Misskey แต่ละแห่งก็มีผู้ดูแลระบบหรือผู้ดำเนินการที่แตกต่างกันไปเช่นกัน

Misskey Project ทำเพียงแค่เผยแพร่แบบแปลนเท่านั้น ไม่ได้เป็นผู้สร้างอพาร์ตเมนต์จริง ไม่ได้เป็นเจ้าของหรือผู้จัดการอพาร์ตเมนต์ และไม่ได้เข้าไปมีส่วนเกี่ยวข้องกับการบริหารจัดการอพาร์ตเมนต์ที่ถูกสร้างขึ้นจากแบบแปลนนั้นแต่อย่างใด

#### อัปเดต

แบบแปลนอพาร์ตเมนต์จะถูกแก้ไขและปรับปรุงอยู่ตลอดเวลา มีการเพิ่มฟีเจอร์ใหม่ๆ เข้าไป (เช่น เพิ่มบันไดหนีไฟ เป็นต้น)

อย่างไรก็ตาม การที่แบบแปลนถูกอัปเดต ไม่ได้หมายความว่าอพาร์ตเมนต์ที่สร้างไปแล้วจะเปลี่ยนแปลงตามโดยอัตโนมัติ

หากต้องการให้อพาร์ตเมนต์เป็นไปตามแบบแปลนรุ่นใหม่ล่าสุด เจ้าของอพาร์ตเมนต์ต้องนำแบบแปลนใหม่มาปรับปรุงอาคารอีกครั้ง ซึ่งก็คือสิ่งที่เรียกว่า “การอัปเดต”

ความถี่ในการปรับปรุงอพาร์ตเมนต์ขึ้นอยู่กับการตัดสินใจตามเจ้าของแต่ละราย ดังนั้น อพาร์ตเมนต์แต่ละแห่งจึงมีความใหม่ไม่เท่ากัน

#### ขอความช่วยเหลือสนับสนุน

ปัญหาที่เกิดขึ้นภายในอพาร์ตเมนต์ เช่น ความขัดแย้งกับเพื่อนบ้าน หรืออุปกรณ์/สิ่งอำนวยความสะดวกชำรุด มีเพียงเจ้าของอพาร์ตเมนต์เท่านั้นที่สามารถจัดการได้

Misskey Project เป็นเพียงสำนักงานออกแบบ จึงไม่มีอำนาจเข้าไปยุ่งเกี่ยวข้องกับอพาร์ตเมนต์

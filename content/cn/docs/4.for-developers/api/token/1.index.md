# 获取访问令牌 (Access Token)

要开始使用 API，必须获取与使用 API 的账号绑定的**访问令牌**。
本文档将先说明获取访问令牌的步骤，然后介绍基本的 API 使用方法。

基本上，API 请求都需要提供访问令牌。
访问令牌是与用户绑定的凭证信息，用于识别使用 API 的用户，同时管理每个访问令牌所拥有的操作权限。

:::tip

用户与其绑定的访问令牌是一对多的关系，即一个用户可以发行多个访问令牌。

:::

您不仅可以轻松获取您自己的访问令牌，还可以获取将要使用您的应用程序的不特定用户的访问令牌。

- 前者的情况：前往\*\*“手动发行自己的访问令牌”\*\*
- 后者的情况：前往\*\*“请求应用程序用户发行访问令牌”\*\*

## 手动发行自己的访问令牌

在 Misskey Web 的“设置 > API”中，可以发行自己的访问令牌。

:::danger

请勿让他人知晓您的访问令牌。

:::

## API 的使用

获取访问令牌后，通过向各类端点 (Endpoint) 发起请求即可使用 API。

:::tip

- HTTP API 全部为 POST 请求，请求和响应均为 JSON 格式（drive/files/create 除外）。
- 在请求头 (Request Header) 中指定 `Content-Type: application/json`。
- 将访问令牌以参数名 `i` 包含在请求体 (Request Body) 的 JSON 中。
- 基础 URL 为 `https://{服务器域名}/api`。

:::

### 使用 Authorization header 的方式

如下所示，指定请求头中的 `Authorization` 字段。

```js
fetch("https://misskey.example/api/notes/create", {
	method: 'POST',
	body: JSON.stringify({
			text: "Hello Misskey API World with My Application!"
	}),
	headers: {
		Authorization: `Bearer ${accessToken}`,
		'Content-Type': 'application/json',
	},
	credentials: 'omit',
});
```

### 使用 i 的方式

包含访问令牌的请求体示例（以请求 meta 为例）：

```json
{
    "i": "HogEFugA1341",
    "detail": false
}
```

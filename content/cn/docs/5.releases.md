## 2025.12.2

发布日期: 2025/12/22

### 注意事项

关于在 v2025.12.0 中进行的“将 config 中 `trustProxy` 的默认值更改为 false”的调整：为了缓解因未根据环境进行正确设置而导致难以登录的问题，我们采取了以下应对措施。

**如果未进行正确设置，可能会导致上述故障，或增加安全风险。请务必检查当前的 config 文件，并根据需要更改该值。**

- 关于 `trustProxy`，在默认情况（config 中未设置该值的状态）下，系统现在会信任环回地址 (Loopback Address) 和局域网 IP 地址空间。
- 补充了关于 `trustProxy` 设置方法的更详细说明。
- 如果希望在反向代理或 CDN 等更上游的层级设置速率限制，或者作为紧急情况下的临时缓解措施，现在可以禁用 Misskey 内部基于 IP 地址的速率限制。

### General

- 依赖项更新

### Client

- Enhance: 添加了 Deck UI 的说明
- Enhance: 添加了防止浏览器清除设置的选项
- Fix: 修复了没有版本声明的 Play 无法正常运行的问题  
  没有版本声明的 Play 将作为 v0.x 系列运行。如果希望在 v1.x 系列中运行，请务必包含版本声明。
- Fix: 修复了在 Deck UI 中将菜单位置设为底部时，无法显示“删除个人资料”按钮的问题
- Fix: 修复了部分 Unicode 表情符号的回应无法变成按钮的问题

### Server

- Enhance: 现在可以禁用 Misskey 内部基于 IP 地址的速率限制
  - 适用于在反向代理或 CDN 等其他层级单独设置速率限制，或用于本地测试等场景。
  - 默认值为 `enableIpRateLimit: true`（启用 Misskey 内部基于 IP 地址的速率限制）。
- Fix: 改善了控制面板中“任务队列”页面所使用的部分 API 的响应速度

## 2025.8.0

发布日期: 2025/08/31

### 注意事项

- 支持的 Node.js 最低版本现已要求为 **22.15.0**。

### General

- 现在删除贴文时，不再同时删除相关联的贴文。
  - 在 API 中，将会出现以前未曾有过的情况，例如“replyId 存在但 reply 为 null”或“renoteId 存在但 renote 为 null”。
- 实现了定期删除旧的远程贴文的功能。
  - 可以在“控制面板 → 性能 → 远程贴文清理 (Remote Notes Cleaning)”中开启。
  - 可以防止数据库过度膨胀。
  - 如果在现有服务器上启用此功能，由于处理量较大，可能会导致存储空间使用量暂时增加。
    - 如果要控制增量，请将最大处理持续时间设置得比默认值更短。
  - 如果对数据库大小没有看到明显效果，请检查 autovacuum 是否已启用。
- 在服务器初始设置完成之前，联邦功能不再开启。
- 日语环境下的可见范围名称“ダイレクト (Direct)”已重命名为“指名”。
  - 这个名称更符合实际的运作方式，即使是不熟悉的人也更容易理解。
  - 其他服务中相当于“私信 (Direct Message)”的 Misskey 功能叫做“聊天 (Chat)”（尽管在旧版 Misskey 中，该功能也被叫做“私信”）。然而，“Direct 贴文”这一名称的存在，曾让人误解这才是私信功能。
    - 未来可能会将“聊天”的名称改回“私信 (Direct Message)”。
- 更新了 mfm.js。
  - Enhance: 支持收录在 Unicode 15.1 和 16.0 中的表情符号。

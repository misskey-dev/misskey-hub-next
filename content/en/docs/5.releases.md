## 2025.12.2

Release date: 2025/12/22

### Note

Regarding the change "Changed the default value of `trustProxy` in config to `false`" introduced in v2025.12.0, the following measures have been implemented to mitigate issues such as difficulties in signing in if the environment is not properly configured.

**Failure to configure this correctly may cause the aforementioned bugs or increase security risks. Please be sure to check your current config and change the value if necessary.**

- For `trustProxy`, the default behavior (when no value is set in the config) is now to trust loopback addresses and local IP address spaces.
- Added more detailed documentation on how to configure `trustProxy`.
- Added the ability to disable the IP address-based rate limit within Misskey. This is useful for cases where you want to set rate limits in upper layers like reverse proxies or CDNs, or as a temporary mitigation during emergencies.

### General

- Updated dependencies

### Server

- Enhance: Added UI descriptions for Deck.
- Enhance: Added an option to prevent settings from being cleared by the browser.
- Fix: Fixed an issue where Play without version notation did not work correctly.  
  Scripts without version notation will run as v0.x.If you want them to run as v1.x, please be sure to include the version notation.
- Fix: Fixed an issue in the Deck UI where the profile deletion button was not displayed when the menu position was set to the bottom.
- Fix: Fixed an issue where reactions with certain Unicode emojis did not act as clickable buttons.

### Server

- Enhance: Made it possible to disable Misskey's internal IP-based rate limiting.
  - This is intended for use when setting up rate limits separately in another layer (like a reverse proxy or CDN) or for local testing purposes.
  - The default is `enableIpRateLimit: true` (Misskey's internal IP-based rate limiting is enabled).
- Fix: Improved the response speed of some APIs used on the Job Queue page in the Control Panel.

## 2025.8.0

### Note

- The minimum supported version of Node.js is now 22.15.0.

### General

- When a note is deleted, related notes are no longer deleted simultaneously.
  - In the API, unprecedented patterns will now appear, such as "reply is `null` even though `replyId` exists" and "renote is `null` even though `renoteId` exists."
- Implemented a feature to periodically delete old remote posts.
  - This can be enabled via Control Panel > Performance > Remote Notes Cleaning.
  - It helps prevent database bloat.
  - If you enable this feature on an existing server, storage usage may temporarily increase due to the large volume of processing.
    - To limit this increase, set the maximum processing duration shorter than the default.
  - If you do not see an effect on the database size, check if `autovacuum` is enabled.
- Federation is no longer turned on until the server's initial setup is complete.

- Feat: When the server is viewed without logging in, content within the server can now be made private.
  - This helps prevent trouble caused by inappropriate remote content (which is hard to moderate fully) being unintentionally published to the internet via your server.
  - You can choose from "Make everything public (previous behavior)", "Make only local content public (= do not publish remote content received within the server)", and "Make nothing public".
  - The default value is "Make only local content public".
- Feat: File types that can be uploaded can now be configured by role.
  - The default includes **text, JSON, images, videos, and audio files**.Other file types like zip are not included, so please change the settings if necessary.
  - In some cases, the file type may not be detected correctly (especially text formats).In that case, the file type is considered `application/octet-stream`.
    - Therefore, if you want to allow these files with unknown types, please add `application/octet-stream` to the allowed list.

## 13.0.0

Release date: 2023/01/16

### TL;DR

- New features (Role system, Misskey Play, New widgets, New charts, 🍪👈, etc)
- Rewritten backend
- Better performance (backend and frontend)
- Various usability improvements
- Various UI tweaks

### Notable features

- Role System:
  - You can manage user policies much more flexibly than before.For example, you can implement rules like "Instance patrons can create up to 30 antennas," "The LTL is hidden by default but visible to permitted users," or "It's an invite-only instance, but any user can invite others." Furthermore, you can combine multiple conditions to automatically assign roles, such as "Local users who created their account less than 1 day ago cannot make public posts."
- Misskey Play
  - A new platform that replaces the traditional dynamic Pages.Specialized for dynamic content (applications), it allows you to create much more flexible applications compared to Pages.

### Changes

#### For server admins

- Node.js 18.x or later is required.
- PostgreSQL 15.x is required.
  - Misskey is not using 15-specific features at 13.0.0, but may do so in the future.
  - Guide for updating PostgreSQL in a Docker environment: https://github.com/misskey-dev/misskey/pull/9641#issue-1536336620
- Elasticsearch support has been removed.
  - Instead, a mechanism to set arbitrary search providers is being planned for the future.You will still be able to use Elasticsearch as before through that mechanism.
- Migrated from Yarn to pnpm.
  Enabling corepack is recommended: `sudo corepack enable`.
- Instance blocks will now also apply to subdomains.
- With the introduction of roles, several features have been integrated into roles:
  - Moderators are integrated into roles.Previous moderator information will be lost, so please record your moderator list beforehand, create a Moderator role after the update, and reassign them.
  - Silence is integrated into roles.Previous users will be pardoned, so we recommend recording the silence list beforehand.
  - Per-user Drive capacity settings are integrated into roles.
  - Instance default Drive capacity settings are integrated into roles.After updating, please edit the Drive capacity in the base role or a conditional role.
  - LTL/GTL open states are integrated into roles.
- Docker execution is no longer performed as `root`.If you are using Docker without object storage, please run `chown -hR 991.991 ./files`.  
  https://github.com/misskey-dev/misskey/pull/9560

#### For users

- The note watch feature has been removed.
- Notifications are no longer created when your poll receives a vote.
- Math formula embedding in notes has been removed.
- It is no longer possible to create new dynamic Pages.
  - Instead, the Misskey Play feature has been implemented, allowing you to create more flexible dynamic content using AiScript.
- AiScript has been updated to 0.12.2.
  - For 0.12.x changes: https://github.com/syuilo/aiscript/blob/master/CHANGELOG.md#0120
  - Plugins older than 0.12.x can no longer be loaded.
- Devices running iOS 15 or below are no longer supported.
- Firefox 110 or below is no longer supported.
  - Even on 109, you can use it without issues by enabling the ContainerQueries flag

#### For app developers

- API: The `meta` response no longer includes the `emojis` property.
  - To fetch the list of custom emojis, request the `emojis` endpoint.
- API: The custom emoji entity no longer includes the `url` property.
  - To display an emoji image, request `<instance host>/emoji/<emoji name>.webp` and the image will be returned.
  - e.g. `https://p1.a9z.dev/emoji/misskey.webp`
  - remote: `https://p1.a9z.dev/emoji/syuilo_birth_present@mk.f72u.net.webp`
- API: The `user` and `note` entities no longer include the `emojis` property.
- API: The `user` entity no longer includes `avatarColor` and `bannerColor` properties.
- API: The `instance` entity no longer includes `latestStatus`, `lastCommunicatedAt`, and `latestRequestSentAt` properties.
- API: `caughtAt` in the `instance` entity has been renamed to `firstRetrievedAt`.

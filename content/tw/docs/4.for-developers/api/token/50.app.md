# 透過建立應用程式取得存取權杖的方法（舊方法）

這裡將說明在導入 [MiAuth](./miauth.md)（v12.27.0）與導入 [OAuth](./oauth.md)（v2023.9.0）之前的傳統存取權杖取得方法。  
若伺服器的版本低於 12.27.0，則必須使用此傳統方式。

## 1. 建立應用程式

將資訊發送至 [`app/create` 端點](endpoints/app/create.html)，以取得 `appSecret`。

```json
{
    // 應用程式名稱
    "name": "test",
    // 應用程式說明
    "description": "my test application",
    // 應用程式權限
    "permission": ["write:notes"]
}
```

此時，若在請求中包含您偏好的 `callbackUrl`，當接下來的存取授權操作完成後，系統會將 `token` 包含在查詢字串中並回呼至該網址。

## 2. 讓使用者進行認證

將 `appSecret` 透過 POST 發送至 [`auth/session/generate` 端點](endpoints/auth/session/generate)。

```json
{
    "appSecret": "fAb12cD34Ef56gH78Ij16kL32Mn64oPf"
}
```

系統會回傳 `token`（這裡假設為 `798b9f6e-248d-43a7-a919-fabc664027f1`）與 `url`。請先使用網頁瀏覽器開啟此 url，並選擇「允許存取」。

## 3. 查詢 accessToken

確認步驟 2 已完成後，將 `appSecret` 與剛剛取得的 `token` 透過 POST 發送至 [`auth/session/userkey` 端點](endpoints/auth/session/userkey)。

```json
{
  "appSecret": "fAb12cD34Ef56gH78Ij16kL32Mn64oPf",
  "token": "798b9f6e-248d-43a7-a919-fabc664027f1"
}
```

在此取得的字串稱為 `accessToken`。請注意，`accessToken` 只能取得一次。

## 4. 產生 i

若是使用 Node.js，可以透過以下程式碼來產生 `i`。這與在設定畫面中直接取得的格式不同，這裡產生的 `i` 會是 64 位數的十六進位字串。

```javascript
const crypto = require("crypto")
const i = crypto.createHash("sha256")
    .update(accessToken + appSecret, "utf8")
    .digest("hex")
console.log(i)
```

## 5. 實際進行測試

```javascript
fetch("https://misskey.example/api/notes/create", {
    method: 'POST',
    body: JSON.stringify({
        i: "/* 在此輸入產生的 i */",
        text: "Hello Misskey API World with My Application!"
    }),
    headers: {
        'Content-Type': 'application/json',
    },
	credentials: 'omit',
});
```
